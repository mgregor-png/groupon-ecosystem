Current funnel: 2.5M monthly US sessions → ~1.8% register (email only, 540K/yr) → 12% purchase within 12mo → $3.83 net value per registration.

Key blocker: No user login on Coupons. Today we only collect emails — no account system, no login, no profile. We need to integrate Groupon's existing auth system into coupons pages, plus build coupons-specific features (saved codes, followed brands, Bucks balance).
Phased implementation:
PhaseWhatEffortWhat You Get

Recommended approach: Start with Phase 1 (Lightweight) to validate conversion. "Sign in with Groupon to claim your $5 gift card." This is the minimum viable test — 2-3 weeks to build, answers the critical question: will coupons users register if the incentive is strong enough?

Why gift cards are the right incentive:
  • Coupons users have extremely high intent and low patience — they came for a code, not to create an account
  • Generic "sign up for updates" will fail. The value must be concrete and immediate.
  • $5 Groupon gift card is the simplest high-value offer. Direct exchange: identity for money. And it pulls users into the Groupon app to redeem — same bridge mechanic as Bucks.
  • Rewards gating (require login for best codes) is an alternative — already tested at +1,247% uplift (Rebecca's ROI model, rewards gating test on NA) — but riskier for high-traffic monetized pages.

Groupon Bucks: The Connective Tissue
Every interaction connects through one mechanism — a visible, growing balance that gives users a reason to come back even when they're not shopping. Bucks accumulate from coupon use, cashback, code verifications, streaks, and referrals. They're spent on core Groupon deals. This creates a shared currency between the affiliate coupon world and the core deals world — without it, the two sides don't reinforce each other.

"You've earned $14.20 this month" — users check their balance even when they're not shopping. That's the habit loop.
The pull-back loop:
User shops on Nike → Groupon overlay shows code → Earns Groupon Bucks → Opens app to redeem → Books a local deal → Earns more Bucks
How it works (iOS 15+): Apps can bundle Safari Web Extensions. User has Groupon app → enables extension in Settings (one-time) → Groupon overlay appears on every shopping site. This is exactly how Honey (inside PayPal app since 2021) and Rakuten work on iOS today. Every Groupon app install becomes a Safari extension install for free.

Hooks to pull users from off-platform back to the app:
  • Coupon auto-apply — best codes surfaced at checkout (affiliate commission)
  • Cashback in Bucks — % back on purchases, paid as Bucks only redeemable in app
  • Brand following — "New Nike code dropped" → push notification → opens app
  • Price watchdog — "Best time to shop Amazon" alerts → opens app to see deals
  • Weekly digest — best codes + "You've earned $X this week, $Y available to spend on deals"
  • Bucks balance visibility — the balance itself is the daily-open trigger. Expiry (90 days) creates urgency to redeem.
Why Groupon wins: Honey/Rakuten bridge coupons → retail rewards. Groupon bridges coupons → local experiences. "Earn Bucks from Nike, spend them on a massage near you." No competitor has this.

Validated by Genie: Same mechanic, $6/user/yr proven on desktop. Safari inherits the economics + the entire app user base (8-10M, not just coupons visitors).

Key unknown: iOS App Store extension bundling. 2-week eng spike validates before any build.

Cashback — how it differs from rewards, risks & engagement layers
Cashback vs Rewards — different mechanics:
• Rewards (Layer 1) = incentive to register. Gift card / Bucks for creating an account. One-time acquisition cost.
• Cashback (this path) = incentive to buy again. Earn % back on every purchase. Ongoing retention cost.
Core mechanic: User buys a Groupon deal → earns 3-5% back as Groupon Bucks → Bucks balance grows → user returns to spend Bucks on next deal. Visible balance creates open habit. 90-day expiry drives urgency.

Cross-platform: Works for ALL app users (iOS + Android). No extension, no registration friction. Leverages existing purchase flow.

Engagement layers (can stack on top of cashback):
  • Daily Deal Drop + Streaks — curated deals rotate daily, streak = bonus (Temu: 89% interaction)
  • Watch This Deal — price/availability alerts (Honey's Droplist model)
  • Geo-Push — "The spa next door has 40% off today" (Shopkick model)
Key risk: Margin erosion. 5% cashback on a $50 deal = $2.50 cost. Must validate frequency uplift > cashback cost. Viktor's L4 data (2.5x CVR, 5.5x frequency when matched to intent) suggests the upside is there for the right segments.

Dependency: Requires Groupon Bucks infrastructure (shared with Paths A and B). Scoping this is Ask #3.

Proposed Tests

Test 2 Paths to Validate Unit Economics

Each test answers: "What is the actual $/user for this path?" Once we have real conversion rates, we can make a data-driven allocation decision.

Test 1: Safari Extension (Layer 2b — Extend Mobile)

What: 2-week iOS engineering spike to validate technical feasibility (App Store bundling, extension permissions). If feasible, soft-launch to 5% of iOS users.

Hypothesis: If we ship a Safari extension inside the Groupon app, at least 10% of iOS users will activate it (PayPal Honey achieves ~30% activation from its app — 10% is conservative), generating >$4/user/yr in affiliate commission + Bucks.

Measures: Activation rate, coupon usage, commission/user, Bucks earning rate, app re-open rate.

Kill gate: If activation <5% or commission <$2/user/yr after 8 weeks → pause and reassess.
Test 2: Registration + Rewards (Layer 1 — Convert)

Context: No user login on Coupons today — only email collection. Users are high-intent and resist friction. The incentive must be concrete and immediate.

What: A/B test two registration incentives on non-monetized coupons pages (zero revenue risk): (1) Rewards gift card ($5 Groupon gift card for sign-up) and (2) Rewards gating (register to reveal best codes). Test on top 100 high-traffic non-monetized pages.

Hypothesis: A high-value incentive (gift card) will increase registration rate from 1% to 3%+ without significant bounce rate increase. Rewards gating already showed +1,247% uplift (Rebecca's ROI model, rewards gating test on NA) in prior test.

Measures: Registration rate per variant, bounce rate change, cost per registration (gift card cost vs $3.83 LTV), 30/60/90-day purchase conversion.

Kill gate: If registration rate <1.5% or cost per registration exceeds LTV ($3.83) or bounce rate increases >20% → revert.

Layer 2a (Genie) continues scaling independently — new platform placement is rolling out now. Layer 3 (Cashback) pilots once Groupon Bucks infrastructure is scoped — it builds on the identity layer from Test 2.
