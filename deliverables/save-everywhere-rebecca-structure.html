The Problem

The Groupon app has a retention problem. Coupons content can solve it.

66% of users buy once and stop. The app only gets opened when someone wants to buy something. We need to give users a reason to come back between purchases.

Our solution

Coupons already drives 29.6M sessions/yr, 14.8M clicks and $12.6M in commission - but it's stuck on the web, disconnected from the app. People need promo codes constantly. By moving this into a Safari extension inside the Groupon app, we turn a web asset into a frequency engine - capturing users off-platform and bridging them back via Groupon Bucks. The core mechanism

Safari Extension Inside the Groupon App

Nobody remembers to open Groupon at checkout. So don't make them. Ship a Safari extension inside the Groupon app - just like Honey lives inside PayPal.

Since iOS 15, apps can ship a Safari Web Extension inside their bundle. The user enables it once in Settings -> Safari -> Extensions and grants website access. Two taps, done forever. From then on, Groupon is there at every checkout - showing available codes and cashback rates. No separate install. No separate app. This is exactly how Honey (bundled inside PayPal since Sept 2021) and Rakuten both work on iOS today.

Why this changes everything:

  • Solves the "remember to open Groupon" problem completely
  • Every activation is a Groupon brand touchpoint - even if the code isn't used
  • Two incentive layers: promo codes (instant discount) + cashback (rewards) - stacks the value
  • Genie already proves the model on desktop - this brings it to mobile without a separate product
  • Turns every Groupon app install into a Genie mobile install for free
How the extension works under the hood
  • Content script injected into Safari pages detects merchant domains (matching against Groupon's coupon database)
  • On checkout/cart pages, it injects a floating banner at the bottom of the viewport - non-intrusive, dismissible
  • Tapping "Apply" pastes the code into the promo field on the page (same approach as Honey)
  • "Claim Bucks" button uses a universal link to deep-link into the Groupon app with the savings amount pre-loaded
  • The app lands on a success screen showing Bucks earned + a "Follow this store" prompt + local deals
Key constraints
  • Safari only - doesn't work in Chrome, Firefox, or in-app browsers (~55% of iOS mobile browsing uses Safari)
  • Content Security Policy - some sites with strict CSP may limit injected UI; the extension should gracefully fall back to Safari's native popup sheet
  • App Store guidelines - Section 4.4 says extensions can't include "marketing or advertising," but Honey and Rakuten are approved, so coupon-finding is accepted

Section 3

Groupon Bucks: The Connective Tissue

Every interaction connects through one mechanism - a visible, growing balance that gives users a reason to come back even when they're not shopping. Bucks accumulate from coupon use, cashback, code verifications, streaks, and referrals. They're spent on core Groupon deals. This creates a shared currency between the affiliate coupon world and the core deals world - without it, the two sides don't reinforce each other.

"You've earned $14.20 this month" - users check their balance even when they're not shopping. That's the habit loop.
Open question: Bucks economics

The value of Bucks as a retention tool depends on the soft currency economy we design. Key decisions still needed:

  • Earn rate: What % of cashback commission do we convert to Bucks? Too low = no incentive. Too high = margin erosion.
  • Redemption ratio: What's a Buck worth in deal credit? 1:1 dollar value, or discounted?
  • Breakage assumption: What % of Bucks are never redeemed? (Industry gift card breakage is ~15-20%)
  • CAC comparison: What does re-engaging a lapsed user cost today via push/email/ads? Bucks need to beat that.
  • Expiry policy: Rolling 12-month expiry creates urgency without feeling punitive

This is a unit economics exercise we need to model before launch - the extension can ship with coupons only while we get the Bucks math right.

Section 4

Unknowns & Risks

The 2-week iOS engineering spike answers the biggest question. The rest are measurable through testing.

What we need to validate

help_outline

iOS App Store - can we bundle it? Honey (PayPal) and Rakuten bundle Safari extensions inside their apps today, but Groupon's app structure may differ. The 2-week spike validates this before any build commitment. Should include a test submission to confirm review timeline and any guideline changes.

help_outline

Extension activation rate: Estimated 10-30% range. Honey achieves ~30% activation from the PayPal app. 10% is our conservative floor. Actual rate determines whether this is a $4M or $16M opportunity.

help_outline

New onboarding flow required: Existing users need to be guided through enabling the extension (Settings -> Safari -> Extensions). This is a new permission step that doesn't exist today. The onboarding UX will directly impact activation rate.

help_outline

iOS share of app users: Estimated ~55% based on general market data. Need confirmed numbers from Radomir/analytics to refine the addressable base.

help_outline

Monthly app install/update rates: New installs get the extension automatically. Existing users need an app update. Update velocity affects ramp timeline.

help_outline

Bucks unit economics: Earn rate, redemption ratio, breakage, and expiry policy all need modelling before Bucks can launch. The extension can ship with coupons-only while this gets figured out.

Technical constraints

block

Safari only: Doesn't work in Chrome, Firefox, or in-app browsers. ~55% of iOS mobile browsing uses Safari. This is a platform constraint, not a bug - but it caps our addressable market on day one.

security

Content Security Policy: Some sites with strict CSP may limit injected UI. The extension should gracefully fall back to Safari's native popup sheet rather than breaking.

gavel

App Store guidelines: Section 4.4 says extensions can't include "marketing or advertising," but Honey and Rakuten are approved, so coupon-finding is accepted practice.

Section 8

The Coupons Space

The broader coupons landscape is shifting. Three concurrent trends create both urgency and opportunity.

trending_down

Organic Decline

Coupons traffic is down 17% YoY. The pure-affiliate model (traffic to codes to commission) is structurally declining - RetailMeNot is down 18% on the same trend. We can't SEO our way out of this.

person_add

Registration as Value Driver

540K annual registrations from coupons pages. Each registration is worth $3.83 in downstream value ($31.88 LTV at 12% conversion). Gating rewards behind registration showed +1,247% sign-up uplift in testing.

extension

Genie Extension + Rewards

Genie currently has 981 installs generating $6/user/yr. With rewards integration (34 MD eng), we project 8-18K installs/month and $9.81/user/yr. This is the desktop proof of concept for the mobile Safari play.

The Honey class-action lawsuit (33% code success rate, misleading savings claims) has created a market window. All major competitors - Rakuten (17M users), Capital One Shopping (#2 free Safari app), Coupert (73% code success) - have Safari extensions. We don't.
