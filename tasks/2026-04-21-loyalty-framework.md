## 2. Competitors - tiered structure

Rebecca's first-draft doc already covers six competitors at surface depth. We re-tier by depth and purpose so Friday's output is honest about what is "deep" vs "scan" and so Dusan's Slickdeals priority is explicit.

### Tier 1 - Loyalty reverse engineering (deep dive)
Focus: how do they build habit and retention? What would we copy?

- **Rakuten** - cashback + Safari extension; longest-running player; points partnerships (Amex, Bilt); paid tier (Rakuten+). Quarterly payout as retention lock.
- **Capital One Shopping** - soft-currency model (Shopping Credits = Bucks inspiration); $45 install incentive (LTV signal); gift-card-only redemption.
- **PayPal Honey** - Safari extension leader on iOS (validates our approach); user base collapsed post-2024 scandal (21M to 13M) - what not to do on transparency.

### Tier 2 - Loyalty scan (lighter pass)
Included to isolate mechanics that repeat across the category, not for full reverse engineering.

- **Ibotta** - CPG / grocery focus; B2B2C Performance Network (IPN) distribution model is strategically interesting; $342M revenue proves cashback-first can scale.
- **RetailMeNot** - Guaranteed 1% cashback floor (Feb 2026 move); 100M+ social reach is the acquisition model; mobile Safari extension.
- **Coupert** - post-Honey challenger; $1 first-withdrawal minimum (low-friction hook); dual currency (cash + Gold Coins).

### Tier 3 - Partnership angle (separate workstream)
Per Dusan - explicit priority, Adam re-flagged Yesterday 11:08 CET.

- **Slickdeals** - UGC deal-hunting community, not a loyalty program. Reverse-engineering loyalty mechanics here is apples-to-oranges (Rebecca flagged). BUT: Dusan sees Slickdeals as likely #1 partner for **replacing Goods on core**. So we treat it as a partnership feasibility analysis, not a loyalty deep dive.
  - Model, traffic, audience overlap with Groupon
  - Partnership economics (revenue share, ad, affiliate)
  - What UGC adds that we can't build internally

**Open for Dusan:** add big loyalty players without a Safari extension (e.g. Starbucks, Marriott Bonvoy) to isolate loyalty from browser mechanics? Rebecca draft currently doesn't include any.

## 3. Research questions - the ~20 points per Tier 1 competitor

Tier 2 answers a subset (~8) at scan depth. Tier 3 (Slickdeals) answers a different question set (partnership fit).

### Signup + onboarding
1. Signup flow step-by-step (screenshots)
2. Signup bonus mechanics and value
3. Onboarding email cadence + content (7-14 day window)
4. Push notification strategy (frequency, triggers, copy)

### Loyalty mechanics
5. Reward currency - cash, points, soft currency
6. Earn mechanics - merchant-funded vs platform-funded
7. Redemption friction - minimum, payout method, speed
8. Breakage assumption (industry signal: ~50%)
9. Tiering / premium paid tier (if any)
10. Partnership currency swaps (Rakuten-Amex-Bilt pattern)

### Acquisition + growth
11. Safari extension UX - when/how it appears (Nike test case)
12. Traffic sources (SimilarWeb)
13. Acquisition channels (paid, organic, app store, influencer)
14. 12-month sign-up / user-base trajectory
15. Cost-per-install signal (Capital One's $45 is the anchor)

### Retention + economics
16. Session / return proxies (SimilarWeb avg duration, pages/visit)
17. LTV estimate (public filings, earnings calls, processed via AI)
18. Commission-at-loss behaviour (Dusan: some run at $0 to build habit)
19. App install + update adoption rate
20. Key differentiator vs others

Rebecca's Apr 20 question: should her team start pulling specifics like app-update adoption curve (50%/80% cutoffs) now or wait for approval? **Proposal:** yes, start on points 11, 12, 14, 16, 19 in parallel - they are bounded and useful regardless of final scope.
